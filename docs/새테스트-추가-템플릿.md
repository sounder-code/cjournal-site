# 새 테스트 추가 템플릿 (쿠루)

이 문서는 `kuru.co.kr` 아래에 새 테스트를 붙일 때 사용하는 최소 작업 템플릿입니다.

## 0) 브랜치
- `git checkout -b codex/<새테스트-슬러그>`

## 1) 경로/파일 생성
- 새 경로: `/<slug>/`
- 파일 기본 세트:
  - `/<slug>/index.html`
  - `/src`를 쓰지 않는 현재 구조라면 루트 JS/CSS 파일 추가

## 2) 필수 메타/공유 태그
- `og:title`, `og:description`, `og:image`
- `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`
- 기본 공유 이미지는 우선 `/og-image.svg` 사용

## 3) 공통 스니펫
- GTM head 스니펫 삽입
- GTM body(noscript) 스니펫 삽입
- `config.js` 로드

## 4) 허브(메인) 연결
- `/index.html` 테스트 목록 카드 추가
- `/home.js`에 카드 클릭/유입 이벤트 연결(필요 시)

## 5) 분석 이벤트 규칙 (권장)
- `start_test` : 테스트 시작
- `analyze_success` : 결과 노출
- `share_click` : 공유 버튼 클릭
- `share_kakao_success` : 카카오 공유 성공

## 6) 광고/정책 체크
- 광고 영역은 본문 흐름을 방해하지 않게 상/하단 배치
- 개인정보처리방침/이용약관/문의 링크 노출
- robots/sitemap 반영 필요 시 갱신

## 7) 배포 체크리스트
- `./npmw run build`
- `git add . && git commit -m "feat: <새 테스트명> 추가"`
- `git push`
- Cloudflare Pages 배포 성공 확인
- 실서비스 URL 수동 테스트(모바일 포함)

## 8) 복붙용 작업 지시문 (새 스레드용)
```
새 테스트 <테스트명>을 /<slug>/ 경로로 추가해.
docs/새테스트-추가-템플릿.md 순서대로 구현하고,
GTM/GA4/카카오 공유/OG 메타까지 포함해서 배포 가능한 상태로 마무리해.
```
