# 발행 자동화 운영 기준

## 일정

- 승인 콘텐츠: 매주 화요일·금요일 05:10(Asia/Seoul), 대기열에서 한 편만 발행
- K-apt 공공데이터: 매월 12일 05:20(Asia/Seoul), 일괄 공개자료 갱신
- 코드 및 데이터가 `main`에 반영되면 CI 검증과 Docker 이미지 발행이 이어진다.

## 콘텐츠 발행 원칙

자동 생성된 원고는 사이트에 바로 공개하지 않는다. 편집 검수를 마친 원고만 `content-queue/`에 넣는다.
예약 원고에는 기존 게시물 메타데이터와 함께 아래 항목이 필요하다.

```yaml
editorialReview: true
autoGenerated: false
publishAfter: '2026-07-14T05:00:00+09:00'
```

본문은 공백과 마크다운 문법을 제외하고 1,800자 이상이어야 하며, 실제 이미지가 한 장 이상 연결돼야 한다.
조건을 만족하는 원고 중 파일명 순서가 가장 앞선 한 편만 발행한다. 대기 원고가 없으면 정상 종료한다.

## 데이터 반영 원칙

K-apt 기본정보·면적정보·관리비정보의 기준일이 일치할 때만 변환한다. 기존 데이터와 비교해 단지 수,
분석 가능 단지 수 또는 관리비 행 수가 비정상적으로 감소하거나 중복·면적 충돌이 생기면 커밋하지 않는다.
검증을 모두 통과하고 데이터가 실제로 달라진 경우에만 `main`에 반영한다.
