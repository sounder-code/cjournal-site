# 새 독립 사이트 배포 가이드 (Cloudflare Pages)

기존 `kuru.co.kr`와 분리된 새 사이트를 같은 레포에서 운영하는 방법입니다.

## 1) 코드 기준점
- 새 사이트 소스: `new-service/`
- 전용 Vite 설정: `vite.new-service.config.js`
- 전용 빌드 출력: `dist-new-service/`

## 2) 로컬 점검
```bash
./npmw run dev:new-service
./npmw run build:new-service
```

## 3) Pages 프로젝트를 새로 생성
Cloudflare 대시보드에서 기존 프로젝트와 별도로 새 Pages 프로젝트를 만듭니다.

빌드 설정:
- Framework preset: `None` 또는 `Vite`
- Build command: `npm run build:new-service`
- Build output directory: `dist-new-service`
- Root directory: 비워두기 (레포 루트)

## 4) 새 도메인 연결
Pages 프로젝트의 `Custom domains`에서 새 도메인을 연결합니다.
예: `new-domain.com`, `www.new-domain.com`

## 5) HTTPS/보안
- `SSL/TLS` > `Always Use HTTPS` 활성화
- 필요 시 `Security` > WAF 기본 규칙 활성화

## 6) 운영 팁
- 기존 허브(`kuru.co.kr`)와 배포 프로젝트를 분리하면 장애 전파를 줄일 수 있습니다.
- 새 사이트에서만 필요한 환경변수/API만 연결해 최소 권한으로 운영하세요.
