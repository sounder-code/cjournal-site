# 개발 실행 프로세스 (Codex 작업 기준)

이 문서는 `나와 닮은 도마뱀 찾기` 프로젝트의 기본 실행 규칙이다.
앞으로 기능 개발/수정/리팩터링은 이 프로세스를 기본으로 진행한다.

## 0) 목표
- 빠르게 배포 가능한 품질로 반복 개발
- 프론트/백엔드/AI/운영을 단계적으로 분리
- 매 작업마다 "작동 확인 가능한 결과물"을 남김

## 1) 기본 원칙
- 한 번에 큰 변경 대신 작은 단위로 쪼개서 반영
- 모든 작업은 "요구사항 -> 구현 -> 검증 -> 기록" 순서로 진행
- 검증 없이 완료 처리하지 않음
- 키/토큰/개인정보는 코드에 하드코딩하지 않음

## 2) 작업 사이클 (반복)
1. 작업 정의
- 이번 턴의 범위를 1~2개 기능으로 제한
- 완료 조건(Definition of Done)을 먼저 확정

2. 영향 범위 확인
- 수정 파일/연관 기능/회귀 위험 식별
- 필요한 경우 기존 코드 먼저 정독

3. 구현
- UI/로직/API를 최소 변경으로 구현
- 공통 로직은 함수로 분리

4. 검증
- 수동 테스트: 업로드, 분석, 결과 노출, 공유 버튼
- 에러 케이스: 이미지 아님, 취소, 네트워크 실패
- 가능하면 lint/test 실행

5. 기록
- 변경 파일 목록
- 사용자 영향 사항
- 다음 작업 1~2개 제안

## 3) 우선순위 규칙
1. 동작 안정성(깨지지 않기)
2. 사용자 핵심 플로우(업로드 -> 결과 -> 공유)
3. 데이터 정확도(매칭 품질)
4. 성능/운영 고도화
5. 부가 기능

## 4) 단계별 실행 로드맵

### Phase A. 프론트 안정화 (현재~단기)
- 로딩/에러/재시도 UX
- 결과 카드 고도화(Top3 + 설명)
- 카카오 공유 실연동 + 인스타 복사 UX 개선
- OG 메타/썸네일 정리

완료 기준:
- 주요 브라우저에서 업로드/분석/공유 동작
- 공유 실패 시 fallback 정상

### Phase B. 백엔드 연결
- `/analyze` API 도입
- 프론트의 분석 로직을 API 호출로 전환
- 결과 ID 기반 공유 링크(`/result/:id`) 구현

완료 기준:
- 프론트가 로컬 계산이 아닌 API 결과 사용
- 링크 재접속 시 동일 결과 조회 가능

### Phase C. AI 매칭 고도화
- 특징 추출(얼굴/색감/분위기) 파이프라인
- 도마뱀 프로필 데이터셋 확장
- 점수 산식 개선 + 버전 관리

완료 기준:
- v1 대비 만족도/재미 지표 개선
- 모델/규칙 롤백 경로 확보

### Phase D. 운영/보안
- 업로드 파일 검증, rate limit, CORS
- 로그/모니터링/에러 알림
- 개인정보/이용약관 반영

완료 기준:
- 운영 장애 감지 가능
- 보안/정책 기본선 충족
